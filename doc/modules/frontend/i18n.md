# i18n（国际化）

最后更新：2026-06-27

## 概述

前端文案系统是自定义的单一中文字典实现（非 next-intl），当前仅支持中文。

## 设计目标

- 按命名空间拆分字典，与功能模块对齐
- 组件直接从 `@/lib/i18n/dictionaries` 导入统一字典
- TypeScript 类型安全：字典结构有 `DeepStringRecord` 类型约束

## 架构与关键文件

| 文件 | 职责 |
|---|---|
| `src/lib/i18n/dictionaries.ts` | 字典组装：9 个命名空间合并为单一 `dictionary`，并提供 `formatMessage` / `translateEnum` |
| `src/lib/i18n/dictionaries/*.ts` | 9 个命名空间字典文件 |

### 命名空间字典

| 文件 | 覆盖领域 |
|---|---|
| `shared.ts` | 通用文案 |
| `enums.ts` | 枚举值翻译 |
| `dashboard.ts` | 仪表盘 |
| `markets.ts` | 市场 |
| `rewards.ts` | 奖励 |
| `copytrade.ts` | 跟单 |
| `ops.ts` | 操作文案 |
| `wallet-analysis.ts` | 钱包分析 |
| `funding.ts` | Polymarket 入金 |

## 核心 API

### Dictionary 类型

`DeepStringRecord<typeof _dictionary>` — 递归映射类型，保证所有叶节点为 string。

### formatMessage / translateEnum

- `formatMessage(template, values?)`：执行 `{key}` 字符串插值
- `translateEnum(value)`：从 `dictionary.enums` 读取枚举中文文案，缺失时格式化原值

## 使用模式

```typescript
import { dictionary, formatMessage, translateEnum } from "@/lib/i18n/dictionaries";

dictionary.rewards.quoteBidRank;
formatMessage("订单 {id}", { id: "123" });
translateEnum("open");
```

- 9 个命名空间字典已实现
- Rewards 字典中的账户余额文案描述 worker 同步的 Polymarket 资金钱包 pUSD 余额，并将 account_capital_usd 标为策略资金上限，避免误解为真实钱包余额
- Rewards 报价配置文案使用“挂单档位 / 买一 / 买二 / 买三”，不再使用“报价偏移”，也不再展示已失效的“单市场额度 / 单腿金额”配置；`requote_drift_cents` 提示描述带确认窗口、冷却和限速保护的换价流程，而不是立即全量撤单重挂
- Rewards 成交后策略文案说明 `exit_at_markup` 的退出加价相对被吃买单原价计算，并说明“持有并续挂”会按被吃买单原价挂卖后继续正常报价
- Rewards 市场筛选文案覆盖最低流动性、24h 成交量、剩余结算时间、Gamma spread 和数据新鲜度门槛
- Rewards 低竞争 sleeve 文案覆盖竞争、奖励、定价/事件风险、退出能力、盘口稳定性和仓位六组解释；配置文案包含低竞争专属报价档位、安全边际、spread、市场评分、入场退出滑点、坏成交恢复天数、top-of-book 跳变、AI allow、信息风险等级和可配置撤单阈值；常见 reject/report reason 已映射成中文卡点。
- Rewards AI 建议和信息风险文案描述 worker 低频/异步模型判断、缓存复用、provider/request format 配置、web search 开关和 API key 仅来自 worker 环境变量；报价计划表列名包含 `AI 建议` 和 `信息风险`；顶部概览包含每日大模型调用统计相关标签
- Rewards 刷新失败会显示明确反馈；live 执行、外部盘口深度和 CLOB 成本精度提示与当前后端行为一致
- Copytrade 字典已收敛为只读钱包跟踪/分析文案，不再出现模拟资金、跟单订单、撤单或重置操作
- 旧 radar/signals/positions/risk 页面字典已移除；Rewards 和 wallet-analysis 的内部持仓/风险文案保留在各自命名空间内
- 通用导航和 `ops` 字典已移除前端 `/replay` 页面文案
- 通用导航已新增 `/funding` Polymarket 入金入口；Funding 字典覆盖后端资金钱包 Polygon USDC/USDT Bridge 入金表单、链上余额、余额查询失败提示、发送前复核和安全边界文案，当前纯内网部署不再展示二次确认码提示
- 通用顶栏字典包含移动端导航菜单按钮的可访问文案
- 当前只有中文，不提供 locale cookie、运行时切换或英文回退
