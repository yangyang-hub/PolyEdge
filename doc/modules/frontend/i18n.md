# i18n（国际化）

最后更新：2026-07-11

## 概述

前端文案系统是自定义的单一中文字典实现（非 next-intl）。当前只支持中文，不提供 locale cookie、运行时切换或英文回退。

## 设计目标

- 文案按命名空间拆分，与功能模块和共享 UI 对齐。
- 组件从 `@/lib/i18n/dictionaries` 导入统一 `dictionary`。
- 字典结构通过 `DeepStringRecord` 约束，叶子节点保持 string。

## 架构与关键文件

| 文件 | 职责 |
|---|---|
| `src/lib/i18n/dictionaries.ts` | 导入并合并字典，导出 `dictionary`、`formatMessage`、`translateEnum` |
| `src/lib/i18n/dictionaries/shared.ts` | 通用导航、布局、状态文案 |
| `src/lib/i18n/dictionaries/enums.ts` | 枚举值翻译 |
| `src/lib/i18n/dictionaries/dashboard.ts` | 仪表盘 |
| `src/lib/i18n/dictionaries/markets.ts` | 市场、事件和证据相关文案 |
| `src/lib/i18n/dictionaries/rewards.ts` | LP rewards 页面 |
| `src/lib/i18n/dictionaries/ops.ts` | 通用操作文案 |
| `src/lib/i18n/dictionaries/funding.ts` | Funding 页面 |

## 核心 API

```typescript
import { dictionary, formatMessage, translateEnum } from "@/lib/i18n/dictionaries";

dictionary.rewards.quoteBidRank;
formatMessage("订单 {id}", { id: "123" });
translateEnum("open");
```

- `formatMessage(template, values?)`：执行 `{key}` 字符串插值。
- `translateEnum(value)`：从 `dictionary.enums` 读取枚举文案，缺失时把下划线替换为空格。

## 当前状态

- 当前合并 7 个命名空间字典。
- 控制台导航只覆盖 `dashboard`、`markets`、`events`、`rewards`、`funding`、`settings`。
- Rewards 文案覆盖 live LP 策略、Run/Cancel/Reset 命令、市场筛选、机会评分、报价构造、成交后合并、盘口选择、AI advisory、信息风险、事件窗口、库存控制、资金和订单/持仓/成交/事件表格。
- Rewards 文案将账户余额描述为 worker 同步的 Polymarket 资金钱包 pUSD 余额，将 `account_capital_usd` 描述为策略资金上限。
- Rewards 文案明确区分 maker 成本目标和受控损失风险 floor，说明成交后保留互补 BUY、动态 rank、库存偏斜、单市场预算、全局 resting-BUY 暴露和非对称 requote。
- Rewards provider 文案使用 allow/reduce/stop-new/evidence cancel 语义，并说明 API key、base URL、模型名和 web search 开关来自 worker 环境变量，不进入前端配置。
- Rewards Runs/Decision Analytics 已补齐 run/action/readiness 状态、blocker/provider/action 分布与汇总指标文案，组件不硬编码中文枚举。
- Funding 字典覆盖后端资金钱包 Polygon USDC/USDT Bridge 入金表单、链上余额、余额查询失败提示、发送前复核和安全边界文案。
- 通用顶栏字典包含移动端导航菜单按钮可访问文案。

## 修改检查清单

- [ ] 新增文案时放入对应命名空间字典。
- [ ] 新增功能模块时创建新的命名空间字典文件，并在 `dictionaries.ts` 合并。
- [ ] 新增枚举翻译时更新 `enums.ts`。
- [ ] 不在组件中硬编码中英文文案。
- [ ] 修改后运行 `yarn build` 或 TypeScript 检查。
